Indicaciones de proyecto
Sistema de Gestión de Pacientes en un Hospital 
Este proyecto busca diseñar un sistema en C++ para gestionar los pacientes y las citas médicas en un hospital. El programa aplicará estructuras dinámicas y recursivas para controlar listas de pacientes, citas pendientes y un historial de atenciones. El sistema debe utilizar listas, pilas, colas, TDA, recursividad y memoria dinámica para mantener el registro actualizado y optimizar el uso de recursos. 
Componentes que debe incluir el sistema:
      1. Módulo de Pacientes (Lista Dinámica): 
            - Registrar nuevos pacientes con información como nombre, edad, diagnóstico y número de historia clínica. 
            - Buscar pacientes por nombre o número de historia. 
            - Eliminar registros de pacientes dados de alta. 
      2. Módulo de Citas (Cola): 
            - Registrar las solicitudes de atención médica. 
            - Atender las citas en el orden de llegada (FIFO). 
      3. Módulo de Historial (Pila): 
            - Guardar las atenciones ya realizadas. 
            - Permitir deshacer la úl ma atención registrada. 
      4. Recursividad: 
            - Calcular el promedio de edad de los pacientes atendidos. 
            - Contar cuántos pacientes enen el mismo diagnós co. 
      5. Sobrecarga de Operadores: 
            - Comparar pacientes según su edad o nivel de prioridad médica. 
      6. Ges ón de Memoria: 
            - Utilizar new y delete para crear y liberar memoria dinámica. 
      7. Menú Principal: 
            - Presentar un menú simple con todas las opciones del sistema.
